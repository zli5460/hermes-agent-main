# 不死鸟 V5.1.3 — 客户安装一页纸（防晕版）

给新人：**不用啃完整仓库**，按顺序做完下面三段即可。

---

## 一、这是不是「自动安装」？

**算「一键脚本自动装」，不算「零点击」**——因为 **API Key 不能替你猜**，必须在终端里由客户自己输入（脚本 **不回显** Key）。

| 步骤 | 谁做 | 说明 |
|------|------|------|
| 解压 zip | 客户 | 随便放哪都行 |
| **先装好 Hermes Agent** | 客户 | 没有 Hermes，不死鸟只是插件，装不上 |
| **终端执行** `bash install.sh`（Mac/Linux）或 `install.ps1`（Windows） | 客户 | **这一条就是「一键安装入口」** |
| 跟着脚本回答问题 | 客户 | 默认模型、兜底模型、URL、Key |
| 备份 / 拷文件 / 同步插件 / 皮肤 / 写配置 | **脚本自动** | |
| **`doctor.py --fix` + `doctor.py --verify`** | **脚本自动** | |
| **`hermes gateway restart`（或 start）** | **脚本自动** | **不配网关，改了配置也不生效** |

所以：**交付时要明确告诉客户——安装 = 解压后双击终端进目录，一条命令跑到底；网关脚本会替你重启。**  
若有人说「没有一键」，多半是 **没装 Hermes** 或 **不敢开终端**——交付侧要给「复制粘贴三条命令」级别的演示。

## 二、脑子别晕：只要记住「三层模型」

安装脚本问你的东西，本质上就三类（先搞懂再填）：

### 1）默认模型（日常 primary）

- **干什么用**：平时闲聊、普通问答，**默认连着哪家模型**。  
- **填什么**：你在三方控制台里 **最常用的那个模型名** + **Base URL** + **Key**。  
- **脚本里叫什么**：日常默认模型 primary、日常 Base URL、日常 Key（写入 `.env`，不回显）。

### 2）兜底模型（fallback）

- **干什么用**：**默认模型挂了**（欠费、502、超时、model not found）时，**自动换谁来顶上**，免得对话直接死。  
- **可以和默认同一个网关**，也可以填 **更便宜的备用模型** 或 **本地 Ollama**。  
- **脚本里叫什么**：兜底模型、兜底 URL（可回车复用日常）、兜底 Key（可回车复用日常）。

### 3）高档模型（手动口令，不是安装时「日常会用」）

- **干什么用**：只有用户打了 **`/深度` `/大神` `/真神`** 并 **确认** 之后才会走；平时 **不会** 因为聊天内容自动上去。  
- **脚本里会问**：深度 / 大神 / 真神用的 **模型名**、高档聚合 **Base URL**、高档 **Key**（可与日常同一把，看你供应商）。  

再多一档 **emergency**：脚本按配置写在 `phoenix_open` 里，属于 **「兜底再失败」时的最后一跳**，安装对话里主要体现为 **fallback/emergency 链**，新人可先当成「再兜底一层」。

**一句话记**：  
**默认 = 平时走路；兜底 = 脚崴了换鞋；高档 = 你自己喊口令才去爬山。**

## 三、和你们说的「手脑分离」怎么对齐？

| 说法 | 在不死鸟里大致对应 |
|------|-------------------|
| **脑** | 路由策略（手动口令 + 确认）、记忆、压缩、自愈决策逻辑 |
| **手** | Hermes 真正发起请求、`switch_model`、工具、Gateway |

安装只是把 **「手」接到客户自己的 API**，并把 **「脑」的配置写进本机**；**自愈 / 记忆** 在安装完成后继续在后台跑（脚本会打开记忆、压缩、检查点等 Hermes 侧开关）。

---

## 四、交付同事给客户的最短命令（复制用）

**前置**：已安装 Hermes，`hermes --help` 能出正常帮助。

```bash
cd /你解压的路径/phoenix-v5.1.3-release   # 以实际解压名为准
bash install.sh
```

装完脚本末尾会提示测：**你好 → 深度学习是什么？→ `/深度 …`**（第二条不应弹高档确认）。

再看网关：

```bash
hermes gateway status
```

## 五、还不能做什么（避免过度承诺）

- **全自动无人值守安装**（不交 Key）：除非客户事先写好 `.env` 再由你自己的运维脚本调用——**可以把「无人值守」做成进阶交付**，标准 zip 仍以交互安装为准。  
- **替客户决定用哪家三方**：必须客户自己有 Key。
