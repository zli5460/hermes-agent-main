# Phoenix V5.1.3 初心回归 — 架构与交付蓝图

版本：**V5.1.3** 蓝图 v1.0  
日期：2026-05-09  
定位：不死鸟 Phoenix 在 **「手动挡路由」** 方向上的正式产品叙事与技术对齐文档  
目标读者：小爷、核心开发者、交付同学、审计 / 学员（节选章节）

**说明**：本文与 `Phoenix_V5.1_FreeRoute_升级蓝图.md` **同属蓝图级**，但 **FreeRoute 侧重「语义自动路由 + LM 判档」**；**V5.1.3 已落地的交付取向是「手动挡三口令 + 确认 + 真切换 + 托底」**。两文档并存：**FreeRoute = 远期能力全集；V5.1.3 = 当前对外交付口径与已实现边界**。

---

# 0. 先说结论

不死鸟 V5.1.3 这一阶段要交的，不是「再堆一层聪明路由」，而是先把 **可控性、可审计性、可安装性** 做实：

> **贵模型不偷偷跑；手动口令才升档；确认后才切换；切换必须真切（client 重建）；失败按档 fallback/emergency；记忆 / 自愈 / 压缩在底下照常运转；Hermes 本体入口不被劫持；配置权威落在 `phoenix_open`。**

建议中文定位：

## **不死鸟 V5.1.3 — 初心回归版**

英文对内代号：**ManualTier + AutoFallback**（手动挡档位 + 自动托底）

核心目标（对客户可讲）：

| 维度 | 要什么 |
|------|--------|
| **可控** | 闲聊 **不会** 自动升到 Claude / GPT 高价档；只有 **`/深度` `/大神` `/真神`** 才进入高档流程 |
| **透明** | 高档前 **确认 / 降级 / 取消**；不搞「假切换」 |
| **省心安危** | 主模型失败 → **档内** fallback → emergency；不等同「全城乱换模型」 |
| **可交付** | `install.sh` / `install.ps1` + `doctor.py` 闭环；静态验收脚本可重复跑 |
| **不毁底座** | **`hermes` 命令永远是 Hermes Agent**；Phoenix 只做插件 |
| **密钥** | 包内 **无开发者 Key**；Key 进客户本机 `.env` |

# 2. 为什么要从「自动路由」退半步做「手动挡」？

## 2.1 现场真实痛点（与 FreeRoute 文档 BUG 列表同源）

- **配置双轨**：旧 `router.task_modes` 与 `phoenix_open` 并存时，**学员以为配的是 A，实际跑的是 B**。  
- **假切换**：只改 `model` 字符串，**provider/base_url/client 未切** → 404 / 502 / model not found。  
- **关键词误触发**：「深度学习」「分析一下」触发贵模型 → **_cost与信任**一起崩。  
- **入口被劫持**：错误教程把 Phoenix CLI 叫 `hermes` → **终端一级入口全毁**。

## 2.2 V5.1.3 的产品裁决

```text
宁可「不够聪明」，也要「绝不冤枉花钱」。
```

因此 **高档触发权交给用户显式口令**，而不是交给 LLM 猜意图（猜错了就要骂人）。

> **日常聊天**：走 Hermes 默认模型（客户自备三方或官方）；**不搞自动猜你要 Claude**。  
> **要强**：用户自己打 **`/深度`** 等；系统只负责 **确认 + 真切 + 托底 + 事后回到默认**。

# 3. V5.1.3 总架构（脑 / 手分离不改）

```text
┌─────────────────────────────────────────────────────────────┐
│                        用户输入                               │
│           CLI / TUI / Telegram / 其它 Hermes 通道             │
└────────────────────────────┬──────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────┐
│   Phoenix 插件（phoenix_full）消息拦截层                     │
│   • 识别 /深度 /大神 /真神（精确前缀规则，避免日常误触）        │
│   • 非口令 → 不擅自升档                                       │
└────────────────────────────┬──────────────────────────────────┘
                             │
              ┌──────────────┴──────────────┐
              │ 高档口令？                 │
              └──────────────┬──────────────┘
                     NO │           │ YES
                        ▼           ▼
           ┌────────────────┐  ┌─────────────────────────────┐
           │ 维持 Hermes   │  │ ConfirmGate：确认 /降级 /取消 │
           │ 当前默认模型   │  └──────────────┬──────────────┘
           │ 走日常对话     │                 ▼
           └────────────────┘  ┌─────────────────────────────┐
                             │ ProviderResolver             │
                             │ 解析 model+provider+url+key   │
                             └──────────────┬──────────────┘
                                            ▼
                             ┌─────────────────────────────┐
                             │ Hermes switch_model()       │
                             │ api_mode = chat_completions   │
                             └──────────────┬──────────────┘
                                            ▼
                             ┌─────────────────────────────┐
                             │ 执行任务 / 工具 / Gateway    │
                             └──────────────┬──────────────┘
                                            ▼
                             ┌─────────────────────────────┐
                             │ 失败则档内 fallback/emergency │
                             │ （ Plugin 内编排尝试顺序）     │
                             └──────────────┬──────────────┘
                                            ▼
                             ┌─────────────────────────────┐
                             │ 结束后回到日常默认模型策略     │
                             └─────────────────────────────┘
```

**脑（路由决策）**：只做「是否触发口令」「是否需要确认」「解析到哪一档」—— **不靠闲聊语义替你下决心**。  
**手（执行）**：Hermes 原生执行 + 真 `switch_model`。

# 4. 「自愈 / 记忆 / 压缩」——在这一版里扮演什么角色？

这些子系统在 **`~/phoenix` 运行时目录**里延续（与 FreeRoute 文档中的「长期能力」一致），**不因手动挡而关闭**：

| 模块 | V5.1.3 口径 |
|------|-------------|
| **记忆** | `memory` 配置保留；会话摘要 / 项目记忆按既有策略工作；**不与「手动路由」冲突**——路由只管「这一轮任务用哪档模型」。 |
| **自愈 self_heal** | 故障抗体、进化等仍在代码树中；**高档失败后优先走配置好的 fallback/emergency**，不等于「无声乱换全局默认模型」。 |
| **压缩 / checkpoint** | 控制上下文体积与快照；减 **413 / 爆上下文** 风险，与路由手动挡 **正交**。 |
| **成本 / credit_monitor** | 监控与告警可继续启用；**手动挡减少意外烧高端**是根本手段，监控是第二层。 |

对外话术：**「路由手动挡管花钱的最前一公里；记忆和自愈管任务连续性与稳定性。」**

# 5. 五档模型（V5.1.3 实际行为表）

与 FreeRoute「五档全自动建议」不同，**V5.1.3 交付包推荐口径**如下：

| 档位 | 触发 | 自动升档？ | 确认？ | 备注 |
|------|------|------------|--------|------|
| **daily** | 默认对话 | **否**（不向高档自动跳） | 否 | 客户安装时写入 daily 模型 |
| **medium** | 预留 `/中等`（可选） | **否**（默认 enabled 可关） | 按配置 | V5.1.3 默认偏「关自动跳出来」 |
| **deep** | **`/深度 …`** | **否** | **是** | 手动 + 确认 |
| **god** | **`/大神 …`** | **否** | **是** | 手动 + 确认 |
| **super_god** | **`/真神 …`** | **否** | **是** | 手动 + 确认；可有 secondary 预留 |

**铁律**：普通句子里的「深度」「分析」「设计」 **单独不构成升档理由**（除非你们另行开通 LM 层——那是 FreeRoute 远期合并项）。

# 6. Hermes 自带的「斜杠切模型」还要吗？

**要。** 那是 **Hermes UI / CLI 自带能力**，用来换 **全局默认模型**。

```text
Hermes：   / 或模型菜单 → 换「默认连着谁」
Phoenix：  /深度 /大神 /真神 → 「这一轮高档任务链」
```

二者 **并行**：Phoenix **不负责**关掉 Hermes；也 **禁止**把 Phoenix CLI 伪装成 `hermes`。

---

# 7. 配置权威：`phoenix_open` + 安装器写入

## 7.1 单一权威源

运行时应以 **`phoenix_open`**（在 `~/.../phoenix/config.json`）为 **Phoenix 档位与 providers** 的权威来源；安装脚本会把安装访谈结果 **合并写入**。

旧版 `router.task_modes` 若仍存在，仅宜作 **迁移参考**，不应再当运行时双轨（历史包袱需在迁移脚本里显式处理）。

## 7.2 Key 与环境变量

- Key **不进仓库**；安装过程写入 **`~/.hermes/.env`**（如 `PHOENIX_DAILY_API_KEY` 等）。  
- **禁止**在聊天窗口收集 Key；**应在终端安装脚本里收集**。

---

# 8. 真切换铁律（与 FreeRoute 一致）

禁止：

```python
return {"model": "claude-sonnet-4.6"}
```

必须：

```text
ProviderResolver → Hermes switch_model(..., api_mode="chat_completions", ...)
→ client 重建 → 再执行任务
```

插件内已对「假 hook 切换」做拦截思路（详见 `plugins/phoenix_full/__init__.py` 注释与实现）。

---

# 9. 安装器与 `hermes` 入口（交付红线）

## 9.1 安装器必须满足的硬条件

1. **安装前后执行 `check_hermes_entrypoint`**：`hermes` **不得**指向 Phoenix 目录、**不得**输出 Phoenix CLI 劫持文案。  
2. **不得**创建名为 `hermes` 的 shim 指向 Phoenix。  
3. **Gateway**：`hermes gateway restart || start` 作为安装收尾之一。

## 9.2 若客户说「装完 hermes 没用」

优先排查：

```bash
which hermes
type hermes    # fish/bash
hermes --help | head
```

- **command not found**：Hermes 未装或 `pipx ensurepath` 未进 PATH。  
- **进了不死鸟**：删错误 **alias / function / 软链**，恢复 Hermes 原生命令。

**这不是 V5.1.3 设计目标，属于错误部署或旧教程遗留。**


负责：插件双路径、皮肤、`config`、网关线索等（以 `doctor.py` 为准）。

## 10.2 发布包内 `sim_verify_v513.py`

交付侧重复跑：**包完整性 / 敏感信息 / install 静态链 / phoenix_open 手动挡语义 / 插件静态审计 / doctor 隔离模拟**。

# 11. 对客户 / 学员怎么说（对外简体话术）

```text
不死鸟这一版，先把「花钱可控」做到位：

• 平时聊天，就用你自己配的默认模型，系统不会偷偷给你升到很贵的那种。
• 只有当你输入「/深度」「/大神」「/真神」并确认后，才会用高档能力。

• 终端里的 hermes 命令应该仍然是 Hermes 本体；如果不是，说明环境被旧教程改坏了，先看 doctor 或重装 Hermes 命令路径。

想换默认用哪个模型，仍然用 Hermes 自带的方式；不死鸟只管这三条「更强任务」口令。
```

# 13. V5.1.3 最终铁律（本版交付）

1. **高档只能靠 `/深度` `/大神` `/真神`，不靠闲聊猜。**  
2. **确认前不执行高档 costly path。**  
3. **切换必须走 `switch_model`，禁止假返回 model 字符串。**  
4. **`phoenix_open` 为 Phoenix 档位权威；避免双轨运行时打架。**  
5. **Key 不进包、不回显、不写日志明文。**  
6. **`hermes` 入口不被 Phoenix 接管。**  
7. **自愈 / 记忆 / 压缩继续服务稳定性，但不替代手动挡的「花钱红线」。**
